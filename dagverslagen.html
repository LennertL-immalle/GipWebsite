Dag verslagen

19 oktober 2020

vWe hebben vandaag al heel wat gedaan. Eerst werd ik geïntroduceert in het bedrijf. Daarna kreeg ik wat uitleg over wat we allemaal gingen doen. Het plan was dat ik een server ging opzetten en al kennis leer maken met de software die Comitor gebruikt. Maar door problemen met de server zelf heeft een hele dag geduurd om deze te laten werken.

Nadat we de server aan de praat hadden gekregen gingen we 3 verschillende virtuele machines maken. Het is de bedoeling dat de 3 virtuele machines met elkaar kunnen communiceren

  1. een Veeam server
  2. een domain controller server
  3. een gitlab server

Ik heb al een korte kennis making gehad met de programmas "Veeam Back-Up & recovery" en "IT-Glue gehad"


20 oktober 2020

Vandaag hebben we voortgezet waarmee we gisteren begonnen zijn. Onze Virtuele machines zijn nu verbonden met een virtuele switch zodat ze met elkaar en de host kunnen communiceren.

Vandaag hebben we ook al het programma Veeam Back-up & replication op de Veeam server gezet. Ik heb al een beetje met Veeam geexperimenteerd. Met dit programma kunne we back-ups maken. Het is een heel uitgebreid en intressant programma.


21 oktober 2020

Nu dat onze virtuele machines klaar zijn voor gebruik kunnen we Veeam Back-Up & Replication gebruiken. We hebben al een back-up job gemaakt. Met behulp van veeam powershell kunnen we de gegevens van onze back up opvragen. Ik heb ook al een klein scriptje geschreven zodat we zoveel mogenlijk info kunnen ophalen van deze backup job.

Ik ben ook aan een meer theoretisch deel bezig dat we jammer genoeg niet in de praktijk kunnen uittesten. We gingen onze documentatie van de back-up job automatisch opslaan in IT-Glue met behulp van een powershell script.

Ik heb ook op de Gitlab server al een Linux distributie gezet. Maar omdat de andere case study nogal veel tijd vraagt gaan we niet verder met gitlab gaan.


22 oktober 2020

Ik heb vandaag een dag mogen meedraaien op de Service desk van Comitor. Dit houdt in dat we klanten helpen met problemen die zij hebben ivm IT. Ik heb mogen zien welke programma's ze allemaal gebruiken om dit zo makkelijk mogenlijk te maken. Via deze programma's kunnen zij direct zien of alles goed gaat met de servers zonder dat de klant hoeft te bellen. Dit heet een ticketing systeem. Als er iets fout gaat wordt er een ticketje gemaakt.

Zij maken ook gebruik van IT-Glue voor documentatie zodat ze de klant zo snel en efficiënt mogenlijk kunnen voorthelpen. Ook gebruiken ze enable om remote alles te kunnen oplossen.

Het was een zeer leerzame en informatieve dag vandaag.


23 oktober 2020

Vandaag was het de laatste stage dag. Ik heb nog een nieuwe case study gemaakt over het opzetten van een FireWall, daarover heb ik veel uitleg gekregen van de mensen die daar werken. Daarnaast heb ik ook foto's getrokken van het bedrijf voor mijn taak van frans. Ik heb ook nog een fietstruitje gekregen van Comitor. Het was een zeer leerrijke en informatieve stage geweest. Dit gaat mij zeker helpen in mijn studiekeuze van volgend jaar.
